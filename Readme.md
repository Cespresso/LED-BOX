# LED-BOX

ESP32-C3ベースのIoTデバイスで、8x8 LEDマトリックスディスプレイをBluetooth Low Energy (BLE)経由で制御できます。

## 機能

### 🔵 Bluetooth Low Energy (BLE) 制御
- **サービス UUID**: `455aa9f0-2999-43de-81b4-54e0de255927`
- **特性 UUID**: `681285a6-247f-48c6-80ad-68c3dce18585`
- **読み書き機能**: BLE特性から8バイトのデータを読み書きして、LEDマトリックスの表示パターンを制御
- **デバイス名**: "LED BOX"
- **セキュリティ**: パスキー認証 (123456)

### 💾 NVS (不揮発性ストレージ)
- LEDパターンや設定をフラッシュメモリに永続保存
- 電源オフ後も設定が保持される
- `KEY_NUM`キーで現在の表示データを管理

### 🔳 8x8 LEDマトリックス制御
- **ドライバ**: MAX7219
- **インターフェース**: SPI
- **ピン配置**:
  - SCLK (クロック): GPIO8
  - CS (チップセレクト): GPIO9  
  - MOSI (データ): GPIO10
- **データフォーマット**: 8バイト配列（各バイトが1行の8ピクセルを表現）

## ハードウェア構成

```
ESP32-C3 → MAX7219 → 8x8 LEDマトリックス
GPIO8   → SCLK
GPIO9   → CS  
GPIO10  → MOSI
```

## 使用方法

### BLE接続でLEDパターンを制御

1. **デバイス接続**: BLEスキャンで"LED BOX"を検索・接続
2. **認証**: パスキー `123456` を入力
3. **データ書き込み**: 特性に8バイトのデータを送信
   ```
   例: [0x00, 0x42, 0x24, 0x18, 0x18, 0x24, 0x42, 0x00]
   ```
4. **データ読み取り**: 現在の表示パターンを取得可能

### デフォルトパターン
8バイト以外のデータを送信した場合、以下のデフォルトパターンが表示されます：
```
0x00 ........
0x42 .#....#.
0x24 ..#..#..
0x18 ...##...
0x18 ...##...
0x24 ..#..#..
0x42 .#....#.
0x00 ........
```

## API詳細

### BLE特性の動作
- **書き込み時**: 受信データをNVSに保存し、LEDマトリックスを更新
- **読み取り時**: NVSから保存されたパターンを返す
- **データ形式**: Raw bytes (最大32バイト、LEDパターンは8バイト)

### LED制御
MAX7219ドライバを使用した8x8マトリックス制御：
- 各行のアドレス: 1-8
- 各行のデータ: 8ビット（1ビット=1LED）
- 輝度制御: 最大輝度 (0x0F)
- スキャン制限: 全桁表示 (0x07)
